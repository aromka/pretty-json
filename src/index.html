Pretty JSON Custom Element <pretty-json>

<pretty-json> allows you to render JSON objects in HTML documents with human readable formatting and expandable interaction for browsing deep JSON structures.

Features

  • It's outputs is a valid JSON string and can be copied and pasted into a JSON file.
  • HTML Custom Element without any dependencies. Works in any modern browser.
  • No need to install any dependencies or build tools. Just drop the script in your HTML and start using it.
  • Allows a high level of customization using CSS variables.
  • Display large JSON objects with expandable and collapsible sections.
  • Supports truncating very large strings and arrays with an ellipsis.

Usage

Include pretty-json.js in your HTML page and use the HTML custom element as the following 

<pretty-json>
{ 
    "hello": "world",
    "value": 42,
    "enabled": true,
    "extra": null,
    "nested": { "key": "value" }
}
</pretty-json>

Your JSON will be rendered as following:

{ "hello": "world", "value": 42, "enabled": true, "extra": null, "nested": { "key": "value" } }

Attributes

You can customize the rendering of the JSON object by setting the following attributes on the <pretty-json> element

expand

By default the JSON object is rendered expanded up to 1 level deep. You can set the expand attribute to a number to expand the JSON object up to that level deep 

<pretty-json expand="2">{ "hello": { "world": "!" } }</pretty-json>
{"hello": {"world": "!"}}

Collapsed by default

You can set the expand attribute to 0 to render the JSON object collapsed by default 

<pretty-json expand="0">{ "hello": { "world": "!" } }</pretty-json>
{"hello": {"world": "!"}}

Complex Example

You can render complex JSON objects with nested objects and arrays. The <pretty-json> element will render them with proper indentation and formatting 

<pretty-json>{
    "hello": "world",
    "value": 42,
    "enabled": true,
    "extra": null,
    "complex": { "key": "value", "obj": { "key": "value" }, "array": [1, 2, 3] },
    "array": [1, 2, 3],
    "deeply": {
        "nested": {
            "object": {
                "with": {
                    "lots": {
                        "of": {
                            "keys": "values"
                        }
                    }
                }
            }
        }
    }
}</pretty-json>
{ "hello": "world", "value": 42, "enabled": true, "extra": null, "complex": { "key": "value", "obj": { "key": "value" }, "array": [1, 2, 3] }, "array": [1, 2, 3], "deeply": { "nested": { "object": { "with": { "lots": { "of": { "keys": "values" } } } } } } }

Primitive Values

Since values like strings, numbers, booleans and null are valid JSON values, they are rendered as such

Strings

<pretty-json>"this is an example string"</pretty-json>
"this is an example string"

Numbers

<pretty-json>42</pretty-json>
42

Booleans

<pretty-json>true</pretty-json>
true

Null

<pretty-json>null</pretty-json>
null

Arrays

Arrays are rendered with [ and ] brackets and each item is rendered on a new line 

<pretty-json>[1, 2, 3]</pretty-json>
[1, 2, 3]
